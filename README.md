# PLC Nova Console

Полная документация ПО для управления PLC-контроллером через COM-порт.

Приложение построено как desktop WebView на Python + Eel:
- backend на Python обрабатывает протокол и работу с COM;
- frontend на HTML/CSS/JS предоставляет современный UI;
- при наличии pystray приложение работает с системным треем.

---

## 1. Назначение и возможности

PLC Nova Console предназначен для замены legacy-утилиты и поддерживает:

- Подключение к контроллеру по COM-порту (реальное устройство).
- Чтение/запись времени и даты (TIME).
- Чтение/запись географических координат и часового пояса (GEONEZ).
- Чтение/запись битовых масок KEYS и MASK.
- Отправку базовых команд COM и raw-команд.
- Управление яркостью, сценариями и сервисными кодами.
- Программирование спектральных каналов и сцен через COMA.
- Назначение групп (Unit3/Unit8 логика).
- Редактор расписания SHED с таблицей, фильтрацией и пресетами дней.
- Генерацию астрономических таблиц (txt/csv).
- Журнал TX/RX-обмена с контроллером и постоянный файл лога.
- Системный трей с пунктами «Открыть» и «Выйти».

## 2. Архитектура

### 2.1 Общая схема

1. Frontend в web-интерфейсе вызывает функции backend через Eel.
2. Backend валидирует параметры и формирует команды протокола.
3. DeviceService отправляет команды в serial-порт, читает ответ и пишет журнал.
4. Frontend показывает результаты в UI и протокольном логе.

### 2.2 Ключевые файлы

- app.py
   - Точка входа приложения.
   - Инициализация Eel, системного трея, логирования, runtime-кэша.
   - Экспорт API-методов для фронтенда.
- backend/config.py
   - Модель конфигурации AppConfig.
   - Загрузка/сохранение config.ini.
- backend/device.py
   - Реальная работа с pyserial.
   - Подключение/отключение COM.
   - Отправка команд, чтение ответов, журнал TX/RX.
   - Поиск COM-портов (pyserial + fallback через реестр/mode).
- web/index.html
   - Разметка UI (вкладки: База, Управление, Расписание, Протокол, Справка).
- web/app.js
   - UI-логика, валидация, вызовы backend API, таймеры повтора сценариев.
- web/styles.css
   - Современная адаптивная стилизация, светлая/темная тема.
- backend/build.txt
   - Пример команды сборки onefile через Nuitka.

## 5. Установка и запуск из исходников

Откройте PowerShell в папке проекта и выполните:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
pip install -r requirements.txt
python app.py
```

После запуска откроется окно UI, а backend начнет слушать локальный Eel-сервер.

## 6. Сборка в EXE (Nuitka)

Пример (из backend/build.txt):

```powershell
.\venv\Scripts\python.exe -m nuitka app.py --onefile --windows-console-mode=disable --output-dir=build --output-filename=PLC_Nova_tray.exe --windows-icon-from-ico="..\app_icon.ico" --include-data-dir=web=web
```

Что делает команда:

- --onefile: единый exe.
- --windows-console-mode=disable: без консольного окна.
- --include-data-dir=web=web: упаковка frontend-ресурсов.
- --windows-icon-from-ico: иконка exe (если файл доступен).

Результат сборки помещается в папку build.

Или с версии 2 используя Pyinstaller - с графической обёрткой auto-py-to-exe:

```
pip install auto-py-to-exe
```

Далее запустите:

```
auto-py-to-exe
```

Откроется окно, во вкладке Settings и нажмите на: Import config from json file, откроется окно выбора, выберите app.json. Внимание! Результат сохранится в папке output в той директории, откуда был запущен auto-py-to-exe, при необходимости отредактируйте пути. Также используется UPX для упаковки ресерсов, при необходимости во вкладке Advanced в поле --upx-dir измените или уберите путь к папке. Также по умолчанию включен лимит рекурсии в 5000, если нужно больше - выключите во вкладке Settings.

## 7. Конфигурация, логи и runtime-файлы

### 7.1 Как вычисляется рабочая директория

- При запуске из исходников используется PROJECT_DIR = родитель папки проекта.
- При запуске собранного exe используется папка, где расположен exe.

### 7.2 config.ini

Хранит:
- COM-порт и baudrate.
- Параметры Admin (Address/Mask/Theme и др.).

Создается автоматически при первом запуске.

### 7.3 commands.ini

- Если файл найден, из него загружаются шаблоны команд для datalist в raw-консоли.
- Поддерживается чтение в кодировке cp1251 (с fallback).

### 7.4 logs/plc_nova.log

- Ротационный лог приложения.
- Содержит API-вызовы, ошибки и trace-строки.
- Основной путь: PROJECT_DIR/logs/plc_nova.log.
- Fallback при ограничении прав: %LOCALAPPDATA%/PLC_Nova/logs/plc_nova.log.

### 7.5 Runtime-кэш

- Используется для Chrome profile/cache при запуске Eel.
- Очищается при старте приложения.
- Основной путь: %LOCALAPPDATA%/PLC_Nova/cache.
- Fallback: PROJECT_DIR/.plc_nova_cache.

### 7.6 Генерация астрономии

Создаются файлы:
- astronony.txt
- astronony.csv

Важно: путь для служебных/генерируемых файлов определяется через PROJECT_DIR в app.py.

## 9. Протокол и команды

### 9.1 Основные команды

- TIME
   - Чтение времени контроллера.
- TIME HH:MM:SS DAY=1..7 DATE=DD.MM.YY
   - Запись времени/даты.
- GEONEZ
   - Чтение координат.
- GEONEZ=LAT,LON,ZONE
   - Запись координат (например 055,037,03).
- KEYS / KEYS=ABC
   - Чтение/запись битов KEYS.
- MASK / MASK=ABC
   - Чтение/запись битов MASK.
- COM AAA,CCC
   - Универсальная адресная команда.
- COMA000,CCC
   - Команды сцен и спектра (без пробела, адрес 000 для broadcast).
- SHED
   - Чтение расписания.
- SHED=...
   - Запись строки расписания.

### 9.2 Адресация AAA

- AI: 1..220 (индивидуальный адрес).
- AG: 0..29 (групповой).
- Преобразование AG в протокол:
   - AG=0 -> AAA=000 (broadcast).
   - AG=1..29 -> AAA=221..249.

### 9.3 COMA: сценарии и спектр

Формат:

```text
COMA000,CCC
```
(адрес 000 = broadcast для всех светильников)

Воспроизведение сцены: CCC = 010 + N, N=1..20.
- Пример: сцена 1 -> COMA000,011
- Пример: сцена 5 -> COMA000,015

Запись сцены: CCC = 030 + N, N=1..20.
- Пример: записать сцену 5 -> COMA000,035

Стартовый сценарий: CCC = 220 -> COMA000,220

Каналы спектра:

- BASE=140: красный
- BASE=160: синий
- BASE=180: дальний красный
- BASE=200: белый

Код канала:

```text
COMA000,BASE+S
```

Где S вычисляется из процента:

- 0% -> S = 0 -> COMA000,140 (красный выкл)
- 1-10% -> S = 1 -> COMA000,141
- 11-100% -> S = процент/5 - 1

Примеры команд:
- красный 50% -> S = 50/5 - 1 = 9 -> COMA000,149
- белый 40% -> S = 40/5 - 1 = 7 -> COMA000,207
- белый на полную (100%) -> S = 100/5 - 1 = 19 -> COMA000,219

Выключение всех светильников (4 команды):
```
COMA000,140 COMA000,160 COMA000,180 COMA000,200
```

### 9.4 SHED (расписание)

Типовой формат:

```text
SHED=09 1 1111111 2-00:10 250 007 0000
```

Поля:

- index: 00..19
- active: 0/1
- days: 7 бит (пн..вс)
- time_type:
   - 0 = фиксированное время
   - 1 = рассвет
   - 2 = закат
- sign: + или -
- hh:mm
- address
- command
- level (опционально, зависит от режима)

## 11. Валидации и ограничения

- COM: baudrate > 0.
- AI: 1..220.
- AG: 0..29.
- Яркость: 0..10.
- Номер сцены: 1..20.
- Сервисный command_code: 230..250 (для яркости/сервисного режима).
- Геоданные:
   - lat: 35..74
   - lon: 20..160
   - zone: 0..12
- Астрономия:
   - year: 2000..2100
   - morning/evening correction: -120..120
- SHED:
   - index: 0..19
   - hours: 0..23
   - minutes: 0..59
   - level: 0..9999

## 12. Диагностика и эксплуатационные заметки

Если есть таймауты или «порт открыт, но ответа нет»:

1. Проверьте драйвер (например CP210x).
2. Проверьте, что выбран правильный COM-порт.
3. Проверьте настройки RS-485, если поддерживаются адаптером.
4. Проверьте полярность A/B и общий GND.
5. Убедитесь в совпадении baudrate с устройством.

Особенности поведения:

- Для части COM/COMA команд используется fire-and-forget (без ожидания ответа).
- При broadcast (AAA=000) некоторые COMA-команды отправляются повторно для надежности.
- Запись сцены в modern UI выполняет legacy-совместимую цепочку:
   1) отправка 4 спектральных каналов;
   2) затем COMA ... 03N.
